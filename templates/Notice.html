🚜 Dashboard Agricole — v12.0

Application web de suivi des interventions agricoles, couplée à Traccar (GPS) et enrichie d'un carnet de terrain, d'un cahier de fertilisation et d'un module analytique.

Prérequis d'installation

Matériel recommandé

  • Pour une utilisation en réseau : NAS Synology ou tout serveur Linux
  • Pour une utilisation monoposte : un PC linux ou windows
  • Accès réseau local et/ou internet pour Traccar et météo

Python

  • Python 3.9 ou supérieur
  • Vérifier l'installation : python3 --version

Dépendances Python

Installez les packages requis via votre terminal :

pip3 install flask requests openpyxl fpdf2
Package Usage
flask Serveur web de l'application
requests Communication avec l'API Traccar
openpyxl Export Excel (.xlsx)
fpdf2 Génération des rapports PDF (analytique, fertilisation)

Les modules suivants sont inclus par défaut dans la bibliothèque standard Python : sqlite3, json, re, threading, csv, io, os, time, secrets.

Traccar

  • Instance Traccar accessible en réseau (locale ou distante)
  • Version Traccar ≥ 5.0 vivement recommandée
  • Les appareils GPS doivent être préalablement déclarés dans Traccar
  • Les périmètres virtuels créées dans Traccar correspondent directement aux parcelles de l'exploitation.

Configuration

Option 1 — Via l'interface (Recommandé, nouveauté v12.0)

Ouvrir le dashboard → ⚙️ État système → section Paramétrage Traccar :

  • Renseignez l'URL du serveur, l'identifiant et le mot de passe utilisateur.
  • Cliquez sur 💾 Enregistrer — la configuration est appliquée immédiatement sans aucun redémarrage.
  • Cliquez sur 🔌 Tester pour vérifier instantanément la bonne liaison avec l'API.

Cette configuration est définitivement sauvegardée dans le fichier config.json et rechargée automatiquement à chaque démarrage du serveur.

Option 2 — Dans le code (Fallback / Secours)

Si le fichier config.json est absent, l'application utilise par défaut les valeurs codées en dur situées dans dashboard.py :

TRACCAR_URL      = "http://votre-traccar:8082/api"
TRACCAR_USER     = "votre@email.com"
TRACCAR_PASSWORD = "votre_mot_de_passe"
DAYS_BACK        = 7    # Période par défaut en jours affichée à l'écran
CACHE_DURATION   = 60   # Durée du cache Traccar en secondes

Lancement

Pour démarrer l'application directement :

python3 dashboard.py

Ou utilisez le script automatisé de redémarrage :

bash restart.sh

L'application est ensuite accessible via votre navigateur à l'adresse : http://[IP-du-serveur]:8080.

Connexion par défaut

  • Identifiant : admin
  • Mot de passe : défini automatiquement lors du premier lancement (consultez le fichier généré password.txt).

⚠️ Pour des raisons de sécurité, il est fortement recommandé de changer le mot de passe via l'onglet Système → Changer le mot de passe dès votre premier accès.

Carnet Phyto et Interventions

Saisie depuis une détection automatique Traccar

  1. Sur une ligne d'événement du tableau principal, cliquez sur le bouton ✍️ (Saisir).
  2. Sélectionnez le type d'intervention (celui-ci est automatiquement déduit en fonction de l'outil attelé).
  3. Renseignez la surface — le champ se pré-remplit automatiquement avec la surface cultivée cadastrale si elle est paramétrée, sinon il reste vide.
  4. Ajoutez les produits utilisés à partir du catalogue (la dose conseillée se pré-remplit seule).
  5. Importez la météo automatiquement 🔄 — le système récupère les données météo historiques exactes au jour et à l'heure de l'intervention (et non du jour actuel).
  6. Validez en cliquant sur Sauvegarder.

💡 Le bouton Modifier (✏️) présent dans l'historique du carnet vous permet de corriger ou modifier a posteriori absolument toutes les interventions (qu'elles soient issues d'une détection GPS ou saisies à la main).

Saisie manuelle

Pour enregistrer les travaux agricoles réalisés sans l'aide du tracker GPS Traccar :

  1. Sélectionnez une ou plusieurs parcelles via les cases à cocher (un champ de recherche textuelle ainsi que des boutons de sélection globale "Tout/Aucun" sont à disposition).
  2. Renseignez la date/heure, le tracteur et l'outil employé. Le type de travaux est déduit seul par rapport à l'outil choisi.
  3. La surface se pré-remplit avec la surface cadastrale uniquement si une seule parcelle est cochée.
  4. Ajoutez les produits et lancez l'importation de la météo historique.
  5. Si vous cochez plusieurs parcelles d'un coup : le système génère intelligemment une intervention distincte pour chaque parcelle.
  6. Saisissez la durée du chantier (en minutes) : elle sera sauvegardée et exploitée pour calculer les rendements en ha/h du module analytique.

Gestion Parcelles et Exploitation

Onglet Parcelles

Champ Description et rôle
Identifiant export Code court (ex: PAR001) servant de passerelle avec la périmètre virtuel Traccar.
Surface cultivée (ha) Valeur pivot servant de référence pour les pré-remplissages automatiques, le registre phytosanitaire (XML) et l'IFT.
Statut Permet de spécifier l'état de l'assolement (En culture, Jachère, etc.).

Note : La surface saisie manuellement au sein d'une intervention reste prioritaire sur la surface cadastrale parcelles pour tous les calculs de l'analytique.

Onglet Exploitation

Renseignez ici les coordonnées administratives requises par les documents officiels :

  • SIRET et Raison sociale de la structure
  • Numéro de Certiphyto de l'exploitant/opérateur
  • Descriptif du matériel roulant
  • Numéros de contrôle réglementaires

Catalogue Produits

Permet d'ajouter ou modifier vos intrants et fournitures :

  • Phytosanitaire : Renseigner obligatoirement le numéro AMM, la dose homologuée, le Délai Avant Récolte (DAR) et le stade BBCH cible.
  • Engrais : Saisir précisément les teneurs pondérales en Azote (N) / Phosphore (P₂O₅) / Potasse (K₂O) (%). Ces chiffres débloquent les calculs automatiques d'unités du cahier de fertilisation.
  • Semences : Indiquer la dose conseillée pour les semis.

📈 Module Analytique

Outil d'audit accessible via l'icône de la barre d'actions ou directement sur /analytique. Il dispose de filtres dynamiques croisés (période, tracteur, outil, type de travaux, parcelle).

Organisation en 3 onglets :

📊 Général

  • Graphiques d'activités par tracteur (surfaces travaillées, débits de chantier en ha/h, nombre de passages), par outil, et par type d'intervention.
  • Graphique d'évolution mensuelle : affiche les surfaces cumulées couplées à la courbe d'ha/h sur un axe secondaire.
  • Tableaux analytiques détaillés par parcelle et par couplage tracteur × outil.
  • Chronologie complète des interventions classée par ordre d'exécution.
  • Calcul automatisé du rendement en ha/h dès que le temps de travail en minutes est présent.

🗺️ Suivi parcellaire

  • Carte thermique parcelles × mois : Visualisez d'un coup d'œil l'intensité du travail du sol ou des traitements par un dégradé de couleurs.
  • Historique par parcelle : Tableau interactif cliquable affichant les détails des opérations de la parcelle.
  • Indicateurs temporels : calcul automatique du délai moyen, minimum et maximum en jours s'étant écoulé entre deux passages.

🌱 Produits & Phyto

  • Bilan volumétrique par produit (surface traitée, volume/quantité totale consommée, dose moyenne appliquée).
  • Indicateur de surdosage (Ratio dose appliquée / dose homologuée) : Représentation visuelle par jauge de couleur sécuritaire (Vert ≤ 100%, Orange ≤ 120%, Rouge > 120% en cas de dépassement de la dose AMM).
  • Listing détaillé et traçabilité des produits épandus par parcelle.

📄 Export PDF Analytique : Génère à la demande un rapport complet de synthèse au format paysage A4 comprenant l'intégralité des graphiques et des tableaux selon les filtres choisis au préalable.

🌿 Cahier de Fertilisation

Accessible sur /fertilisation. Vous permet de piloter vos apports d'engrais à l'aide de filtres par campagne, culture ou parcelle.

⚠️ Prérequis indispensable : Les fiches produits de vos engrais doivent obligatoirement comporter leurs teneurs en N, P₂O₅, K₂O (%) dans le catalogue sous peine de fausser les indicateurs.

Organisation en 3 onglets :

📊 Bilan apports

  • Indicateurs clés (KPIs) : Azote moyen efficace apporté (U/ha), phosphore moyen, potasse moyenne et indicateur de surface totale fertilisée.
  • Graphiques décisionnels : Comparatif de l'Azote (N) par parcelle face à l'objectif fixé, diagramme en anneau de la balance N/P/K, taux de couverture des besoins et graphique du cumul d'N dans le temps.
  • Tableau de suivi des objectifs : Progression en temps réel de vos apports d'azote par parcelle vis-vis du plan de fumure (Vert pour une conformité ≥ 90%, Orange si sous-fertilisation < 90%, Rouge si sur-fertilisation risquée > 110%).
  • Lignes de parcelles cliquables pour déplier instantanément le détail complet de chaque apport (date de l'épandage, produit, dose hectare, unités pures d'N, P, K générées).

🎯 Objectifs

  • Interface permettant de saisir et de moduler vos objectifs de fumure en unités (U/ha) pour l'Azote, le Phosphore (P₂O₅) et la Potasse (K₂O), et ce, par type de culture.
  • Les valeurs saisies sont stockées en base SQLite pour être injectées dans les calculs visuels et les exports d'édition.

📋 Catalogue engrais

  • Vue d'ensemble de la liste des engrais référencés affichant leur formule brute NPK ainsi que les unités d'N/ha qui seraient apportées selon la dose conseillée.
  • Formulaire d'édition de fiches intégré directement sur la page (nom de l'engrais, dose de base, unité de mesure, pourcentages N%, P%, K%).

📄 Export PDF Fertilisation : Génère un document officiel au format portrait A4 imprimable, créant automatiquement une feuille par parcelle avec l'historique complet des apports, le cumul global NPK et l'écart visuel face à l'objectif de rendement.

Bilans IFT et Registre Phyto

Calcul des Bilans IFT

  • Sélectionnez simplement une culture ainsi qu'une campagne de travail.
  • Le système gère nativement le chevauchement des cultures d'hiver (ex : blé, triticale, colza) en nommant les campagnes sous la forme 2025-2026.
  • Affiche les volumes d'intrants totaux, la surface cadastrale de l'îlot et l'Indicateur de Fréquence de Traitement (IFT) propre à chaque produit appliqué.
  • ⚠️ Conformément à la réglementation, la surface exploitée comme base de calcul pour l'IFT est strictement la surface cadastrale enregistrée pour la parcelle.

Export Registre Phyto XML

Génère un fichier de données structurées au format XML, répondant point par point au cahier des charges officiel du registre phytosanitaire :

  • Identité administrative et coordonnées de l'exploitation agricole.
  • Historique des interventions phytosanitaires : produit, dose unitaire, surface cadastrale réglementaire (qui prévaut sur la surface calculée par le tracé GPS Traccar), relevés météo certifiés et nom de l'opérateur ayant réalisé le traitement.

Gestion Automatique de la Météo (API Open-Meteo)

Le dashboard intègre l'API publique Open-Meteo (solution gratuite, open-source et ne nécessitant aucune clé d'activation API personnelle) :

Date de l'intervention saisie Source et type de données météo importées
Moins de 24 heures Importation immédiate des conditions météo en temps réel.
Plus de 24 heures Requête automatique dans les archives météo historiques mondiales à l'heure précise où le travail a été enregistré (historique disponible jusqu'en 1940).

Note : Le serveur hébergeant l'application doit obligatoirement disposer d'un accès à internet pour interroger l'API Open-Meteo et remplir les fiches de carnet.

Notes techniques & Maintenance

  • Migrations de base de données silencieuses : La structure de la base SQLite database.db s'auto-ajuste de manière transparente en tâche de fond. Aucune exécution de script SQL ou action humaine n'est requise lors du déploiement d'une mise à jour applicative.
  • Sécurité des accès : Votre mot de passe administrateur est chiffré de manière irréversible à l'aide d'un algorithme de hachage fort (SHA-256) avant d'être sauvegardé dans le fichier système password.txt.
  • Optimisation du réseau (Cache Traccar) : Le temps de conservation du cache des requêtes API Traccar est modifiable via l'interface afin d'éviter les requêtes superflues vers le serveur GPS et de garantir une fluidité parfaite.
  • Design Mobile-First : L'ensemble des tableaux, graphes et cartes bénéficie d'une interface responsive design fluide, optimisée pour le travail de terrain sur écrans tactiles de smartphones ou tablettes.
  • Politique de Sauvegarde (Backups) : Par mesure de sécurité, la base de données database.db est automatiquement dupliquée dans le répertoire backups/ lors de chaque démarrage du serveur, toutes les 24 heures à heure fixe via un thread d'arrière-plan, ou instantanément à la demande via l'action manuelle dédiée de l'interface (⚙️ État système → Sauvegarder).
